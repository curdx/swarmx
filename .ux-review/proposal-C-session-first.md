## ① 一句话定位与目标心智模型

**定位**：flockmux 是把真实 Claude/Codex CLI 团队装进浏览器的并行任务工作台——选一个项目，开几个会话，每个会话在自己的 worktree 里交付一件事：看着它干、卡了它喊你、干完看 diff 合并归档。

**目标心智模型**（用户需要建立的概念 **恰好 5 个**，全部借用已有隐喻，其余一切是这 5 个概念的属性或实现细节）：

| # | 概念 | 一句话定义（即 UI 里的解释文案） | 映射到现有后端对象 |
|---|------|----------------------------------|--------------------|
| 1 | **项目** Project | 一个代码仓库 + 它的设置（关联仓库、默认 CLI、定时任务） | `workspaces` 表（现"工作空间"）+ `workspace_roots`（现"源码上下文"，降为项目设置） |
| 2 | **会话** Session | 一件要交付的事：一条对话 + 一个隔离分支 + 一支临时 AI 小队 + 一份变更。做完即归档 | `threads` 表（现"方向"）。slug/worktree/isolation 全部下沉为实现细节 |
| 3 | **成员** Member | 会话里干活的 AI 进程。每个会话有一个**队长**（拆解任务、向你汇报）和若干**队员**（领活干完即退场） | orchestrator / worker。"调度/orchestrator/编排器"三名合一为"队长" |
| 4 | **计划** Plan | 队长维护的任务计划：先给你看、你可改可批，之后持续更新进度 | blackboard ledger markdown（现"工作台账"），降级为会话内置顶卡片 |
| 5 | **变更** Changes | 会话产出的代码 diff 与收口动作（审查 → 合并 → 归档） | 现有 `GET …/diff` + `POST …/merge`（含 AI 解冲突），首次获得 UI |

**状态不是第 6 个概念，是会话/成员的一个属性**，但必须统一为五态生命周期徽章（替换"绿点=PTY 活着"这一全产品撒谎根源）：

| 状态 | 视觉 | 判定信号（全部来自服务端事实，禁止"自认"） |
|------|------|---------------------------------------------|
| 启动中 | ◐ 灰 | BootstrapStage 事件：spawned → shim_ready → mcp_ready → prompt_injected（新增事件，原料全在日志，be-model 缺口#1） |
| 工作中 | ● 绿（脉动+当前动作一句话） | first_output/first_message 之后，且 last_activity_at 新鲜 / AgentActivity phase=running |
| 等你 | ⚠ 琥珀 | 队长发给 user 的消息未回 / 计划待批准 / 变更待审查 / 权限确认类 needle 命中 |
| 卡住 | ✕ 红 | needle 命中（Not logged in / 限流）/ shim_exit_code≠0 / 首响看门狗 90s 超时 / orphaned handoff / deps gate 300s 超时 |
| 已完成 | ✓ 蓝 | 队长宣告完成或全部队员 farewell，且有变更待收口；合并后 → 已合并 → 归档 |

**核心原则（来自 P0 教训）**：任何 UI 状态词只许描述已被服务端事件证实的事实。绿色只在"收到首条真实输出"后出现；失败是一等 UI 事件，必须出现在对话主视图，而不是埋在抽屉终端的红字里。

| OnboardingTour / Welcome hero / spell 宣传 | **全删**。onboarding = 第一个会话本身（见 3.1） |

## ③ 逐屏说明

### 3.1 入口（首次使用 / 空状态）

没有营销 hero、没有 4 步 tour。第一屏就是用户要做的第一件事，60 秒内发出第一个真任务：

```
┌──────────────────────────────────────────────────────────┐
│                把 AI 团队放进你的项目                      │
│                                                          │
│  项目路径  [ /Users/me/code/my-app            ] [浏览…]  │
│            ✓ 检测到 git 仓库 · pnpm 项目                  │
│            最近：~/code/my-app  ~/work/api   ← 候选 chip  │
│                                                          │
│  环境检查（实时，创建前就跑——失败别让用户进死房间）：       │
│  ✓ Claude Code CLI 已安装 (v1.2.3) · 已登录              │
│  ✕ Codex CLI 未安装  [安装指引]                          │
│                                                          │
│  你想先让 AI 做什么？                                     │
│  [ 总结这个项目的结构并给出 3 个改进建议            ⏎ ]   │
│  示例：「给 X 模块写测试」「修复 lint 报错」               │
│                                                          │
│                                  [创建项目并开始 →]       │
└──────────────────────────────────────────────────────────┘
```

- 项目名默认 = 文件夹名，颜色自动分配（可在项目设置后改）。CreateWizard 的两步弹窗、5 色选择、源码上下文 accordion 全部消失。
- **登录态预检前置到创建之前**：`/api/plugins` 增加 `logged_in/auth_error`（新探针，be-model 缺口#2）。CLI 未登录在这里就拦住并给 `login_command`，而不是让用户进一个永远"暂无消息"的房间。
- 双形态：Tauri 用系统文件夹选择器；浏览器复用已实现的 350ms debounce 路径校验（修复展示位让 ✓/✕ 肉眼可见），文案改为"浏览器版暂不支持系统文件夹选择"，附最近路径候选。

### 3.2 新建会话（prompt-first，⌘⇧N）

不是弹窗向导，是一个内联 composer。提交即进会话，没有任何模态等待。

```
┌─ 新会话 · flockmux-core ────────────────────────────────┐
│ [ 修复 OAuth 回跳后丢失 redirect 参数的问题          ]   │
│                                                         │
│ ▸ 高级（默认折叠）：基于分支 [main ▾] · CLI [Claude ▾]  │
│   · 并行开 [1▾] 份（同题多解挑赢家，Crystal 模式）       │
│                                  [开始 ⏎]               │
└─────────────────────────────────────────────────────────┘
```

- 会话标题由首条 prompt 自动生成（≤24 字摘要），分支名 = 系统自动 slug，只作为会话头的灰色小字徽章展示——**用户不再命名和管理 worktree**。
- 实现映射：createThread（带名 → 后端走现有"命名即隔离"管线建 worktree 并在其中 spawn 队长）→ 立即 navigate 进会话 → 对话流里实时滚出启动清单（3.4）。彻底删除假进度条与 60s 静默超时进房。
- "命名即隔离会杀重启 agent"的 P5-D 副作用从此不再发生在用户眼前：会话创建即命名即隔离，一次成型；后续改标题只改显示名（PATCH name 不再触发重栽——需后端把"重栽"限定在 slug 首次冻结时，属行为收敛非重写）。
- 隔离失败（非 git 等）落 `degraded`：会话头出现常驻琥珀条"隔离失败——正与主目录共享，可能与其他会话互踩 [重试隔离]"（字段已在 API，纯消费）。

├──────────────┬─────────────────────────────────────────────────────────────┤

**对话 ⌘1**（默认块）：
- 顶部「计划」置顶卡 = 现 ledger markdown 的新家，带书写者健康徽章（"队长：活跃 / 已 N 分钟无活动 / 已退出"，last_activity_at 现成可算）。
- 流内是结构化事件卡，不只是消息：队长/队员消息、系统卡（派出队员/队员完成附产出摘要/合并结果）、错误卡（3.4）。派生关系在流里讲成"谁派了谁干什么"，替代协作图。
- 输入框：@成员定向、模型/思考强度（改动会重启队长——改为显式确认句"切换将重启队长，进行中的回复会中断"，消灭隐藏行为）。

**终端 ⌘2**：
- 子 tab：队长 ｜ 各队员（活动中的子 tab 带脉动徽标"在这干活"）｜ 我的终端（人用 shell，cwd=会话 worktree）。
- 每个成员终端 = 现 AgentDrawer 的 xterm + 活动流 + 注入 prompt 条，升级为一等公民；底部状态条从 SPAWN/PTY/HOOK 行话改为 bootstrap 阶段时间线（"spawned ✓ → shim ✓ → mcp ✓ → 注入 ✓ → 首条输出 ✗ 等待 340s"），数据来自新 health 端点。诊断层级越深信号越诚实，绝不允许比浅层乐观。
- **接管 / 交还**：成员终端默认只读；[接管] = 暂停该成员 + 终端变可写（人直接敲 /login 或修环境）；[交还] = 弹一行输入"告诉它你改了什么"作为消息注入。这是浏览器形态下用户替 agent 修登录态的唯一通道，必须有。

**变更 ⌘3**（首次给产出一个家，竞品全员标配而 flockmux 缺位的那一块）：

```
┌ 变更 · fix-login-redirect ← main      3 文件 +84 −12 ┐
│ 文件列表       │  diff（unified/side-by-side）        │
│ ✓ auth.ts      │  - const url = req.query.next        │
│ ✓ auth.test.ts │  + const url = sanitize(req.query…)  │
│ ○ login.tsx    │  [在此行评论 → 派回队长继续改]        │
│────────────────┴──────────────────────────────────────│
│ 收口清单：✓分支有变更 ✓已自动 rebase 检查 ○测试通过   │
│           ○全部文件已查看 ○评论已解决                  │
│ [运行验证]   [合并回 main]（清单未过则置灰附原因）      │
└───────────────────────────────────────────────────────┘
```

- 数据：现有 `GET …/diff`（base/branch/files/base_dirty）起步；行级评论 = 带 文件:行号 上下文的消息发给队长（现有 message API 即可），评论线程随后续 diff 变化标记 resolved。
- 合并：现有 `POST …/merge`；冲突时自动 spawn merge-resolver（现成），UI 呈现为流内系统卡"AI 正在解决 2 处冲突…"。合并成功 → 会话变"已合并" → 提示归档。

### 3.4 状态与失败呈现（P0 的正面解）

**a. 启动清单（替代假进度条 + "暂无消息"）**——创建会话后对话流的第一组内容就是它，每步消费真实事件：

```
┌ 会话正在启动 ─────────────────────────────────────────┐
│ ✓ 创建隔离工作区（分支 fix-login-redirect）      2.1s │
│ ✓ 检查 Claude Code CLI（已安装 · 已登录）        0.8s │
│ ✻ 启动队长并交付任务……                      已等待 6s │
│ ○ 等待队长的第一条回应（通常 40s 内）                 │
└───────────────────────────────────────────────────────┘
```

事件源：步骤① ThreadChanged(state ready/degraded)；②登录探针；③新增 BootstrapStage（spawned/shim_ready/mcp_ready/prompt_injected——把 rest.rs 既有日志升格为 SwarmEvent + `GET /api/agent/:id/health` 快照，WS lossy 所以必须配 REST 快照）；④首条 message/activity。

**b. 失败卡（90s 首响看门狗，服务端兜底）**——任何一步失败或超时，清单原位变成：

```
┌ ✕ 启动失败：Claude Code CLI 未登录 ───────────────────┐
│ 队长终端输出 "Not logged in · Please run /login"      │
│ ▸ 终端最后 10 行（点击展开，不用换页）                 │
│ [打开终端去登录]   [换 Codex 重试]   [重试]            │
└───────────────────────────────────────────────────────┘
```

- needle 检测复用现有 ready_plan 扫描基建，加 "Not logged in"/限流特征串 → AgentState::Error + 结构化原因。
- [换 Codex 重试] 顺带修"静默回退"：spawn 响应加 `fallback_from`，任何回退在流内有系统卡 + 成员 chip 带"已回退"徽章。
- 同一失败同时点亮三处：流内错误卡（主视图）、会话头徽章变 ✕、侧栏会话行变 ✕ + 收件箱角标 +1。**验收标准：拔掉登录态创建会话，90 秒内在对话主视图（非抽屉深处）看到红色失败卡。**

**c. 运行中失败同权**：进房后的失败（限流、队员崩溃、orphaned handoff、依赖等待超时）与启动失败走同一错误卡通道。orphaned handoff warn 升格为事件，渲染为"队员『测试员』在等一个不会出现的信号，可能永久挂起 [让队长检查] [查看详情]"。

**d. "等你"是一等状态**：队长需要决策/授权/登录时，流内插入可操作卡（它想做什么 + 为什么 + 批准/拒绝/看终端），会话行挂琥珀徽章，并发桌面通知（"agent 完成或需要你注意"——Conductor/Crystal 的通知契约）。

**e. 文案诚实化**："AI 引擎就绪"删除；设置页 AI 引擎区分两行事实："已安装 v1.2.3" 与 "登录：✓ 已验证 / ✕ 未登录 [login 指引]"。"N 个 AI 在线"改为"队长正在 <动作> / 队长被卡住"。

### 3.5 并行工作切换（3 项目 × 多会话的扫视与跳转）

- **侧栏行 = 全部扫视信息**：状态点（五态色）+ 标题 + 未读数；"等你"用琥珀 ⚠ 区别于普通未读。项目折叠态聚合角标（⚠1 ●2）。未读数据源改为新的精确未读端点（GROUP BY 一条 SQL，messages 表现成），挂载与 WS 重连时全量校准，去掉 limit 200 近似。
- **收件箱**（顶栏，替代常亮铃铛）：只聚合"等你 / 卡住"的会话，按项目分组，点击直达。进展/心跳类事件永不点亮它。
- **键盘**：⌘K 收录全部"项目 › 会话"条目（带状态徽标）；⌘⇧N 新会话；⌥↑/↓ 同项目内循环会话；⌥⇧↓「跳到下一个需要我的会话」（等你>卡住>未读 优先队列）——重度用户的核心循环"扫一眼→谁需要我→一键过去"每环都有键。
- **切换记忆**：per-session 记住上次所在块（对话/终端/变更），切回不重置。
- **归档**：会话合并或放弃后归档（默认终止其全部成员——顺带堵住"软删空间留幽灵 agent 烧 token"的洞，删除确认框显示"将终止 N 个成员"）。侧栏永远只显示活着的任务；归档进「归档」页可恢复。

### 3.6 双形态与响应式

- 唯一侧栏 <md 收成 Sheet；会话头与成员 chips **任何宽度可见**（删除 hidden 2xl:flex 的成员栏后，状态入口不再随断点蒸发）；三块 tab 文字只有 2 字，窄屏保留文字缩字号，不收成纯图标。
- Tauri：文件夹选择器、系统级桌面通知；浏览器：路径校验 + Notification API。两形态共享全部其余 UI。

## ⑤ 三期落地路线

### 第一期 · 诚实化（1-2 周）——**第一刀切在这里，不动 IA**

后端（纯接线，无 schema 改动）：
1. 新增 `SwarmEvent::BootstrapStage` + `GET /api/agent/:id/health`（阶段+失败原因快照，WS 配 REST 防 lossy）。
2. 登录探针：`/api/plugins` 加 `logged_in/auth_error`；ready_plan needle 加 "Not logged in"/限流串 → AgentState::Error。
3. 首响看门狗：注入后 90s 无 message/activity → 广播可诊断事件（含 orchestrator，消灭看门狗豁免盲区）。
4. spawn 响应加 `fallback_from`。

前端（现有布局内替换）：
5. 聊天空状态 → 启动清单卡 + 失败卡（[打开终端][重试][换 CLI]）。
6. 绿点语义改为"首条输出后"；之前灰色"启动中"；Error 三处同步变红（成员行/状态条/空状态）。
7. 成员栏 <2xl 改为可展开 overlay（临时止血，第二期删除）。
8. 删除空间确认框显示活成员数并默认级联终止。

**验收**：拔登录态建空间，90s 内主视图见红色失败卡；不再存在绿点+“暂无消息”并存的状态。

### 第二期 · 两层 IA 收敛（2 周）

9. 改名落地（项目/会话/队长/计划/推进），唯一术语表 + locale lint；删 16 个死键、孤儿 Context、Welcome hero、tour。
10. 侧栏合并为单栏（项目 › 会话行，五态点+未读）；删成员栏/状态条/统计 chip/重复按钮；成员 chips 进会话头。
11. 会话三块：对话（计划置顶卡 + 结构化系统卡）/ 终端（AgentDrawer 升级为成员子 tab + 我的终端）/ 变更（diff 只读版 + 合并按钮，复用现有 diff/merge）。
12. prompt-first 创建（项目预检前置 + 内联 composer + 自动命名）；PATCH name 与重栽解耦。
13. 精确未读端点 + 侧栏/项目聚合角标；归档流（含终止成员）。

**验收**：新用户从落地到第一条 AI 回复 ≤60s 且全程无新名词弹窗；房间内必遇概念 ≤5。

### 第三期 · 并行与收口闭环（1-2 周）

14. 收件箱 + 桌面通知（等你/卡住/完成）；⌘K 全收录、⌥↑/↓、⌥⇧↓ 跳下一个需要我的。
15. 变更 tab 完全体：行级评论派回队长、收口清单 gate 合并、[运行验证]（项目级 run script）。
16. 终端接管/交还；degraded 警告条 + 重试隔离；orphaned-handoff/wake-state 升格为事件与诊断视图（`GET /api/wake-state`）。
17. 计划卡升级为"可批准的首条计划"（30s 自动放行 + [等我批准] 硬门禁）；项目扫描完成后生成 3 个建议任务 chip。
18. 同题并行 N 份会话；幽灵 agent 清理入口。

**验收**：3 项目 × 6 会话并行时，"谁需要我"可在侧栏/收件箱 O(1) 回答；一个会话从 prompt 到合并归档全程不离开 UI。

## ⑥ 风险与开放问题

- **main 会话的定位**：保留为置顶"主分支会话"兼容现有自动创建逻辑，但长期可能完全隐藏（仅作合并目标）。第二期观察使用率再决定。
- **PATCH name 与重栽解耦**是本方案唯一的后端行为变更（非新增接口），需确认无 spell 依赖该副作用。
- **结构化事件卡的覆盖率**：转录 tailer 的 AgentActivity 粒度是否足以支撑"系统卡"叙事，第二期需实测；不足则先降级为简单文本系统消息。
- **并行 N 份会话**对 token 成本敏感（行业经验 3-5 并行是甜蜜点），创建时显示预估提醒。
