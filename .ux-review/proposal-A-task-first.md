> swarmx 工作空间重设计提案 A。依据：`.ux-review/firsthand-findings.md`（实测）、`analysis-fe-map.md`（前端 IA）、`analysis-be-model.md`(后端能力）、竞品调研（Vibe Kanban / Codex 云端 / Cursor agents / Conductor / Crystal / Jules / Devin / Factory）。

## ① 一句话定位与目标心智模型

### 一句话定位

**swarmx 是一块「AI 任务指挥台」：你派出任务，AI 班组在隔离工作区里完成它，你在任务卡上看到真实状态、在任务页里审查并合并产出。**

进门第一屏不再是"某个工作空间的聊天室"，而是一块跨项目的任务板（进行中 / 等你处理 / 已完成）。agent、分支、终端、聊天全部降级为**任务的属性**。这是 Codex 云端 / Cursor agents / Vibe Kanban 已验证的范式：用户管理的是"要交付的事"，不是基础设施。

### 目标心智模型：恰好 5 个概念

| # | 新概念（唯一中文名 / 唯一英文名） | 定义（用户视角一句话） | 吸收/替换的现有概念 | 后端载体 |
|---|---|---|---|---|
| 1 | **任务 / Task** | 一件要交付的事：一句话目标 + 一个隔离工作区 + 一份产出。有起点有终点（合并或归档） | 方向、thread、worktree、主线、隔离/degraded、全局 /tasks 看板、/goals 目标 | `threads` 表（slug 自动命名）+ `workers` 表（子任务）+ diff/merge API |
| 2 | **项目 / Project** | 任务发生的代码库：一个路径 + 关联仓库 + 默认 AI 引擎 | 工作空间、工作区、源码上下文、挂载、对等项目 | `workspaces` + `workspace_roots` |
| 3 | **班组 / Crew**（队长 + 队员） | 替你干这个任务的 AI：一名队长拆解派工，若干队员执行 | orchestrator/调度/编排器（三名归一为**队长**）、worker（→**队员**）、成员、agent | `agents` + `workers` 表，init spell |
| 4 | **活动流 / Activity** | 任务里发生的一切，按时间排列：你的消息、队长的计划、队员跑的命令、系统事件、错误 | 聊天、工作台账/任务台账/进展状态（→置顶**计划卡**）、协作图、近况/心跳、录像入口 | `messages` + `AgentActivity` + blackboard ledger + 新增 BootstrapStage 事件 |
| 5 | **变更 / Changes** | 任务的产出：与主分支的 diff，审过才能合并 | （现状缺位——4 个视角没有一个展示代码） | 已有 `GET …/threads/:tid/diff` + `POST …/merge`（冲突自动 spawn merge-resolver） |

**概念预算纪律**：首次路径上用户只会遇到这 5 个词。worktree / 分支 / 隔离 / blackboard / wake / handoff / spell / shim / PTY 全部进入"行话防火墙"——用户可见文案禁用，只在任务卡副标题灰字（分支名）、失败诊断详情折叠、终端原文里出现。

**状态语义总纲（P0 的宪法条款）**：任何 UI 状态词只许描述已被服务端事件证实的事实。"进行中"= 已收到该 agent 首条产出之后且活动新鲜；之前一律"启动中"；失败/等待是一等状态，与"进行中"平级展示，永不静默。

```
├─ /notifications                   通知全量页（低频，徽章只数"等你"事件）

```

### 旧全局导航 8 项的去向

| 现状 | 去向 |
|---|---|
| 任务（/tasks worker 看板） | **升格为新首页本体**，但粒度从 worker 改为任务（thread）；worker 卡变成任务详情内"子任务"清单 |
| 目标（/goals） | 并入任务的"验收标准"字段（计划卡内）；过渡期入口收进 ⌘K |
| 终端（人用 shell） | ⌘K「打开终端」+ 项目页工具区；与 agent 终端（任务详情右栏）从此名字不再相撞 |
| 文件 | 项目页工具区 + ⌘K |
| 定时 | 项目设置"定时任务"（本质：定时派任务给队长） |
| MCP / 用量 / 设置 | 合并进 /settings 的 section |
| 通知 | 保留，但红点只对"等你"事件点亮并带数字 |

三套任务体系就此归一：**任务板（thread 粒度）→ 子任务（workers 表，自动推导状态）→ 计划卡（队长的 ledger markdown）**，一条层级、互相内嵌、不再平行。

要点：
- **零营销空状态**。无任务时首屏 = 输入框 + 3 个按项目语言探测生成的 starter 任务 chip（如「总结这个项目的结构」「补全 README」「给 X 模块写测试」）。Welcome hero、3 步引导卡、功能卡片、4 步 tour 全部删除。
- **「等你处理」是收件箱**（Linear Inbox 模式），排序：失败(红) > 等输入(琥珀) > 待审查(蓝)。每张卡自带原因一句话 + 1-2 个就地动作按钮——发现问题、定位原因、一键动作在同一张卡上闭环。
- 卡片信息密度纪律：标题（prompt 自动命名）/ 项目徽章 / 状态语义句 / 班组缩略 / diff 统计 / 未读 ✉。分支名、worktree 路径不出现在卡正面（hover tooltip 展示）。
- 数据源：新增 `GET /api/board`（聚合 workspaces+threads+agent health+精确未读+diff 统计；后端分析③缺口 4 的扩展），WS 增量 + 挂载/重连时 REST 校准（遵守"lossy WS 必须配 REST 快照"纪律）。
- 窄屏（<lg）：三列收为一条分组列表，"等你处理"组永远置顶——核心信号不再像现成员栏那样在 1536px 以下蒸发。

### 3.2 新建：派任务（prompt-first）与新项目

**派任务 = 板顶输入框，一步完成**（Crystal/Jules 模式）：

```
┌ 派出新任务 ──────────────────────────────────────────────────┐
│ 你想让 AI 做什么？                                            │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 修复登录后跳转丢失的问题，补一个回归测试                    │ │
│ └──────────────────────────────────────────────────────────┘ │
│ 项目  [swarmx-web ▾]        ▸ 高级：基于分支 main ▾ · 并行 ×1 │
│                                            [派出 ⏎]           │
└──────────────────────────────────────────────────────────────┘
```

- 任务名由 prompt 自动生成（已有 AI 命名能力），分支/slug 系统起名；base 分支、并行 N 份（同 prompt 开 N 个任务挑赢家，Crystal 模式）收在"高级"里。
- 点「派出」**不弹等待模态**：卡片立刻出现在"进行中"列、带 [启动中] 徽章，点进去看实时启动清单（见 3.4）。CreateWizard 的假进度条与 60s 静默超时从此无处可藏。
- 后端映射：createThread（worktree 隔离）→ init spell spawn 队长 → bootstrap。main 不再是默认落点——main thread 只承载项目页"随聊"。

**新项目**（首次或下拉里选「+ 新项目」）只有一个必填项：

```
┌ 新项目 ─────────────────────────────────────────────────────┐
│ 项目路径  [ /Users/wdx/opc/swarmx-core            ] ✓ 已找到│
│           最近目录: ▸ ~/opc/demo-app ▸ ~/dev/web   （可点选） │
│ ▸ 高级：名称(默认取文件夹名) · 颜色(自动) · 关联仓库(检测到 2 │
│   个本地依赖，可一并挂载)                                     │
│                                              [创建并派任务]   │
└──────────────────────────────────────────────────────────────┘
```

- 双形态：桌面（Tauri）原生文件夹选择器；浏览器模式接通**已存在但未使用的 root-suggestions 端点**和最近目录候选，路径校验（已实现的 350ms debounce）的 ✓/✗ 反馈提级到输入框右侧显著位置；禁用文案改为"浏览器版暂不支持系统文件夹选择"。
- 名称、颜色、关联仓库全部折叠为可选——对齐 Linear"最小必填 + 渐进披露"。

设计决策：
- **聊天降级为活动流的一部分**：用户消息、队长回复、队员的结构化事件（命令/文件改动/错误，可展开）、系统事件（启动步骤、合并、重启原因）合并为一条时间线。"改名方向导致成员闪断"这类隐藏行为（P5-D）从此必须以系统消息形式出现在流里。
- **计划卡 = 工作台账的重生**：ledger markdown 渲染为置顶可折叠卡，标注书写者健康（"队长 · 2 分钟前更新"/"队长已 8 分钟无活动 ⚠"）；/goals 的验收标准成为卡内字段；workers 表渲染为卡内"子任务"行（状态自动推导 + 人工覆盖已有 API）。协作图降级为计划卡的「展开班组关系」视图；"已满足/等待中/派生"改译"可以开工/等上游/谁派出了谁"。
- **变更面板是收口主舞台**（全竞品共识，现状最大缺位）：diff API 已有；行级评论作为带 `文件:行号` 上下文的消息派回队长（复用 sendMessage）；合并按钮以"已查看全部文件"为前置；冲突时走已有 merge-resolver 自动协调并在活动流里直播。合并成功 → 任务自动进"已完成"，提示归档。
- **终端从抽屉升格为一等 tab**（Devin 右栏模式）：每个班组成员一个子 tab，活动中的 tab 带脉动徽标；默认只读，**[接管] 按钮**暂停该 agent 让终端可写（人去敲 /login 或修环境），[交还] 时弹一行输入"告诉队长你改了什么"注入回会话——浏览器模式下用户替 agent 修登录态的唯一通道，必须存在。
- 右栏 <2xl 不蒸发：收为顶栏按钮（带状态色徽章）展开 overlay。
- 「唤醒」改名「催一下」，无确认框（幂等、误点代价≈0），点击后按钮内联变"已提醒 ✓"。

### 3.4 状态与失败呈现（P0 的正面解）

**任务状态机（卡片与详情页共用，全部由服务端事实驱动）**：

| 状态 | 判定来源（全部已有或纯接线新增） |
|---|---|
| 启动中 | thread=preparing，或队长 bootstrap 阶段 < first_output（新 BootstrapStage 事件） |
| 进行中 | 有活 agent 且 last_activity_at 新鲜 / token 用量在涨（API 已返回，前端没算） |
| 等输入 | 未读 to=user 消息；needle 命中交互提示；队长显式提问 |
| 失败 | needle 命中 "Not logged in"/限流；shim_exit_code≠0；`.error` 落黑板；**首响看门狗**：注入后 90s 无 message/activity |
| 待审查 | 队长发出完成信号（handoff/完成消息）且 diff 非空 |
| 已完成 | 已合并 / 已归档（thread 软删，进 /archive 可恢复） |

**启动清单（替换"AI 正在了解你的项目…"黑盒，Conductor setup script / Devin Setup-Agent 模式）**——派出任务后活动流的第一张卡：

```
┌ 任务启动中 ────────────────────────────────────┐
│ ✓ 创建隔离工作区 task/login-fix          (2s)  │
│ ✓ 启动 Claude Code CLI                   (4s)  │
│ ✗ 登录校验失败：Not logged in                  │
│   Claude CLI 未登录。 [打开终端运行 /login]     │
│   [用 Codex 重试]  ▸ 查看原始日志              │
│ ○ 注入任务说明                                  │
│ ○ 等待队长的第一份计划（通常 40s 内）           │
└────────────────────────────────────────────────┘
```

每步消费真实事件（spawned → shim_ready → 登录探针 → mcp_ready → prompt_injected → first_output/first_message），任一步超阈值变黄、失败变红 + 原因 + 就地动作。**失败的任务卡保留在板上**（修掉"createWorkspace 失败回滚删空间、spawn 后失败零兜底"的不对称——失败是一等状态，可重试/换引擎/归档，不消失）。

**配套规则**：
- CLI 静默回退透传：spawn 响应加 `fallback_from`，卡上出现"已回退到 Codex"徽章 + tooltip + 通知。
- "AI 引擎就绪"文案废除；设置页引擎健康分两行："已安装 ✓ / 登录态 ✓|✗（[去登录]）"。
- 删除项目对话框显示活 agent 数并默认勾选"同时终止 N 名成员"；设置-诊断页列出幽灵 agent（已删项目的存活 agent）与孤儿等待（orphaned handoff 升格为事件）。
- 桌面通知（Tauri 系统通知 / 浏览器 Notification API）：任务进入"等你处理"或"失败"时推送，正文 = 任务名 + 原因一句话，点击直达任务页——用户敢离开当前任务去做别的事。

### 3.5 并行工作切换

- **板即切换器**：3 项目 × 多任务的全部状态一屏可扫，O(1) 回答"现在谁卡住了/谁等我"。
- 键盘：`⌘⇧N` 派新任务；`⌥↓/↑` 跳到下一个/上一个"等你处理"的任务（Slack ⌥⇧↓ 模式，最高价值一颗键）；`⌘K` 收录全部任务条目并带状态徽标；任务页内 `⌘1/2/3` 切右栏 变更/终端/录像。
- 每个任务页记住"上次所在面板"（localStorage per-task）；从板进入直达记忆位置。
- 未读数据源：新增精确未读端点（GROUP BY from/thread 的 COUNT，messages 表现成），替换 limit 200 近似值；铃铛红点只数"等你"事件。

### 3.6 项目页（轻量后台）

随聊（main thread，明示"不隔离、直接在主分支目录对话"）+ 任务列表过滤视图 + 设置（路径/关联仓库/默认引擎+登录健康/定时任务）。"源码上下文"更名"关联仓库"，并首次向用户解释真实作用："这些仓库会写进 AI 的上下文文件（CLAUDE.md/AGENTS.md），让队长能看到它们"。

## ⑤ 三期落地路线（每期 1–2 周）

### 第一期（1–2 周）：诚实化——先让状态不撒谎（**第一刀**）

不动 IA，纯接线。这是后续任务状态机的数据地基，单独发布即有价值。

后端（be-model 已确认 80% 是日志升格）：
1. `SwarmEvent::BootstrapStage` + `GET /api/agent/:id/health`（spawned→shim_ready→mcp_ready→prompt_injected→first_output + 失败原因）。
2. ready_plan needle 扩展："Not logged in"/限流/交互提示 → `AgentState::Error` + 结构化原因；`/api/plugins` 加登录态探针。
3. 首响看门狗：注入后 90s 无 message/activity → 广播失败事件。
4. spawn 响应加 `fallback_from`；orphaned handoff warn 升格为事件。

前端（在现 IA 上消费）：
5. CreateWizard 假进度条 → 真实启动清单；超时进房带持久 banner。
6. 聊天空状态 → "队长已启动 Ns，等待第一条消息…"进度卡 / 失败卡（含 [打开终端][重试]）。
7. 成员点三态化（启动中/在线=有首条产出/异常），状态条文案改口；AgentDrawer 底部状态条换 bootstrap 阶段时间线。
8. 删除空间对话框级联终止选项。

验收标准：拔掉登录态重建空间，90 秒内在聊天主视图（非抽屉）看到红色失败说明与一键终端入口。

### 第二期（2 周）：任务详情页成型——单任务体验闭环

1. 新 `/task/:tid` 路由（复用 thread 作用域数据）：活动流（messages + AgentActivity + 系统事件归并；计划卡置顶=ledger + workers 子任务行）。
2. 右栏面板：变更（diff API + 行级评论→sendMessage 派回队长 + 合并按钮/merge-resolver 直播）、终端（per-agent PTY tab + 接管/交还）、录像。
3. 派任务输入框（createThread + 自动命名 + spawn），替换"开新方向"对话框；模型/思考强度改动会重启队长的事实写成活动流系统消息。
4. 术语表全量替换 zh/en 语言包 + 死键清理 + locale lint；删 Welcome spell 卡与 tour。
5. 旧 4 tab 收敛：聊天/台账/协作图/录像 → 活动流 + 右栏（协作图并入计划卡展开）。

### 第三期（1–2 周）：Mission Control 上位——跨项目并行

1. `GET /api/board` 聚合端点 + 精确未读端点；新首页任务板（三列 + 收件箱排序），ChatHome/Welcome 退役。
2. 全局导航收敛：板 / 项目 / 历史 / 设置；goals、tasks、cron、files、终端按 ②表收编；activeWorkspace 隐式联动废除（作用域随任务显式携带）。
3. 归档/历史（thread 软删 + 恢复）、合并后自动归档提示。
4. 通知通道：桌面/浏览器通知 + 铃铛只数"等你"；`⌥↑/↓` 等你队列、`⌘⇧N`、⌘K 收录任务。
5. 项目页（随聊 + 设置 + root-suggestions 接入）；设置-诊断页（幽灵 agent / 孤儿等待 / wake 快照，后端补 `GET /api/wake-state`）。
6. Starter 任务 chip（按项目探测生成）。
