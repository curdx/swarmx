# flockmux 聊天窗口重设计：驾驶舱 / 透明优先

## 1. 一句话主张

**押注「对话归对话、活动归面板」的双区分离**：聊天区只承载人和队长之间干净的目标对话，所有 agent 的机械活动（工具调用、思考、派工、进度、变更）实时流入右侧专用的「工作面板」——多 agent 并行不再是气泡流里的 firehose，而是一个可俯瞰、可钻取、永远诚实的「现在发生什么」空间。

核心赌注：**多 agent 并行本质上需要一个独立的实时工作面板，塞进气泡流必乱**。这与竞品研究里 Devin/Manus/Jules 收敛出的「左对话 + 右工作面 (computer pane)」主流布局一致（rAutonomous 主结论），也是对现状「单流 + 静态成员栏 + 分离 DAG tab」混合方案的纠偏。

## 3. 消息流渲染

**总原则（rMultiAgent 主结论 + 入流律）**：对话区默认只有三种声音——**人、队长、以及精简后的成员交付/求助**。worker 的原始 token 流、worker↔队长 的协调全部进右面板，不进气泡。这样并行 N 个成员，对话区每轮也只多几行，永不 firehose。

### 用户气泡
右对齐、accent 实心、白字、无头像、时间贴底（沿用现状，视觉无需改）。**修掉现状 bug**：用户对自己消息的「回复」入口移除（user→user 回复无语义），回复只在「引用队长某条消息追问」时出现。

### 队长消息（◆ 菱形锚点）
左对齐，**高对比**——纠正现状「队长气泡近乎隐形」这个 P1。队长是 payload，必须是全屏最强元素：深色/accent 描边卡 + 菱形 ◆ 头像（区别于成员圆头像）+「队长」标签。三类内容：
- **计划**：不进气泡，进顶部计划置顶卡（见 §5）。
- **派工宣告**：气泡里一句话「我拆成 3 步，已派给测试和前端」，**详细派工事件进右面板的事件 chip**。
- **综合/答复**：正常 GFM markdown（沿用 ChatMarkdown：代码/preview/mermaid/copy 全保留）。`▸思考摘要` 默认折叠、dim、不混入答案（rPrinciples + rCoding 共识：CoT 降级为暗块，不当结论）。

### 成员（worker）消息（彩色圆头像，稀有入流）
成员**默认不在对话区说话**。只有两种成员事件会冒泡进对话（其余全在右面板）：
- **交付卡**：成员完成时升一张卡——`✓ 测试 完成「写失败用例」`，含**作业证据**（改了 N 文件 +x/−y、测试输出折叠块、`[查看变更]` 跳右面板变更 tab）。回应现状「silent handoff / 无工作证明」缺口。
- **求助/待输入卡**：成员卡住需要人——见 §7。

每个成员有**稳定身份**（role 派生名 + 固定色 + 圆头像），在对话气泡、乐队栏、面板事件 chip 三处复用同一色同一名（rMultiAgent 关键 affordance）。禁用 `worker_7` 这种 ID，用 `测试 / 前端 / 调研` 这类意图名。

### 系统事件卡（居中发丝线 → 升级为结构化卡）
现状把 wake/blackboard 噪音过滤掉是对的，保留。但**有意义的系统事件**（隔离结果、引擎回退、模型切换、启动失败）按入流律必须**双写**：在对话区落一条**紧凑居中卡**（一行原因 + 可选行动），**完整内容进右面板活动流**。例：对话区 `ⓘ 引擎回退 Claude→Codex`，点开在右面板看详情。

### 归属与折叠（防 firehose 的四道闸）
1. **声音过滤**：对话区只显 人/队长/成员交付与求助；worker 原始流不入。
2. **派工/交付作为事件而非气泡**：派工进右面板 chip（可展开队伍小树），不占对话一整段。
3. **同成员连续事件合并**：右面板里同一成员的连续工具调用折成一行 `测试 · 跑用例 (3 步) ▸`，点开展。沿用 AgentDrawer 现有的「finished turns 折成一行」逻辑。
4. **时间分隔 + N 条未读分隔**：沿用现状（>5min 插时间线、首条未读插一条 accent 分隔），不变。

## 4. 「现在发生什么」机制

这是本方向的心脏，专治 doomscrolling gap。落到右面板顶部的**「现在」live 区** + 中部**活动时间线**。

### 「现在」live 区（面板顶置，常驻）
每个**正在干活的成员**一行 live 卡，三个诚实信号（rCoding「两信号诚实状态」+ rAutonomous「1-2s 内首卡」）：
```
● 测试   正在 跑 npm test   ········  12s   ⌃▣
  ▸ Bash: vitest run tests/login.test.ts
```
- **动词**：来自 AgentActivity tailer 的 `label`，翻译成用户动词「正在 跑测试 / 正在 编辑 login.ts / 正在 读文件」（绝不泄漏 worktree/blackboard/wake）。
- **滴答计时 + 静默降级**：last-activity 计时；超过 45s 无新 activity → 行变灰 + `⚠ 已 50s 无输出`（诚实，不假装在跑）。这直接修现状 P1「PendingBubble 不随 agent 死亡消失」——live 卡绑 agentState，agent 死立即移除。
- **`⌃▣` 看终端**：一键把深度面板切到该成员的终端（PTY），钻取最深一层。

**首卡保证（1-2s）**：成员一 spawn，立即在「现在」区落一张 `◐ 测试 正在 启动引擎 1s` 卡——绝不让人盯着绿点猜。这是入流律 + rAutonomous 对「状态撒谎」最直接的修法。

### 活动时间线（面板中部，倒序流）
所有已结束的活动按时间倒序，作为可回放的时间线（rAutonomous「replay timeline」）。每行 = `✓/✕ <成员> <动词> <对象> <耗时>`，同成员连续折叠。这一层吸收掉现状散落在「成员栏 breadcrumbs / AgentDrawer 活动 tab / 协作图」三处的信息，统一成一条流。

### 思考与派工的表达
- **思考**：队长/成员的 thought_trace 不进对话，作为暗色折叠块挂在对应事件下（默认折叠）。
- **派工**：作为彩色事件 chip `✓ 队长 派工→测试: 写失败用例 ▸展开队`，点 `▸` 展开一棵小队伍树（队长→成员、依赖、状态），**取代独立 DAG tab**（回应现状「第 5 视角过载」「DAG 教训」）。
- **进度**：成员的 `<role>.progress.md` 不再塞进窄成员栏（修现状「100KB 卡死 rail」P2），而是作为活动流里的一行进度事件，截断 + `▸展开`。

### 钻取（核心交互）
点乐队栏任一成员 / 点活动流任一行 / 点对话区成员头像 → **面板进入「单成员焦点」模式**：只显该成员的现在 + 活动 + 终端 + 变更，对话区不动。再点「全部」回到全队视图。这是 rMultiAgent「drill into one agent 是一等可逆焦点」的落地，复用现有 AgentDrawer 的五 tab 内容，但**不再是 880px 抢屏抽屉，而是右面板原地换内容**。

## 5. 计划 / 派工 / 交付 / 审批 / diff

明确每类事件**入流（对话）/ 进面板 / 可操控**的归属：

| 事件 | 对话区 | 右面板 | 可操控 |
|---|---|---|---|
| **计划** | 顶部置顶卡 | — | 队长改→卡原地更新；可「批准计划」门控 |
| **派工** | 队长一句话宣告 | 事件 chip + 队伍树 | 点 chip 钻取成员 |
| **交付** | 交付卡(证据) | 变更 tab 累积 diff | `[查看变更]`→面板 |
| **审批/求助** | 求助卡(三要素) | 对应成员现在区高亮 | `[批准][拒绝][看终端]` |
| **diff/变更** | 仅交付卡入口 | 变更 tab(主场) | 行级评论 + 合并门 |

### 计划置顶卡（计划先行）
钉在对话区顶部（不滚走），是 blackboard ledger 的重生。含目标、成功标准、**队长健康**（`工作中● / 已 N 分无活动⚠ / 已退出✕`，回应「写手健康」）、可勾选步骤 `▢调研→进行中 ▢用例 ▢改代码`，随成员完成原地打勾（rAutonomous「live checklist 是最高信号进度条」）。给会话一个可见**终点**，治「无承载物 / 无目的地」。
- **降级保护**（现状 P1）：若队长首条不是计划结构，不阻塞，渲染成普通消息，计划卡留空提示「队长尚未给出计划」——计划卡是渐进增强不是硬依赖。
- **批准门**（可选）：贵的并行跑前，计划卡底部 `[批准并开工]`，治「错计划浪费整轮」（rAutonomous Plan-gate）。

### 交付卡（作业证据）
成员 handoff → 对话区交付卡：`✓ 测试 完成 · 改 1 文件 (+14 −0)` + 测试输出折叠 + `[查看变更]`。点进右面板**变更 tab**，所有成员的 worktree 写入聚合成可评审 diff。

### 审批 / 求助卡（三要素，禁裸按钮）
`①要做什么 ②为什么 ③预期结果 + [批准][拒绝][看终端]`。`[看终端]` 走右面板终端钻取。禁裸 APPROVE/DENY，治橡皮图章。

### diff / 变更（右面板主场）
变更 tab = 文件列表 + side-by-side diff。**门控移到合并前**（rCoding 共识：gate 从 before-write 移到 before-merge，per-file accept 摘樱桃）：
- **行级评论**：点行「在这评审」→ inline「告诉队长改这里」→ 作为带 file:line 上下文的消息发给成员（flockmux 史上第一个评审面）。
- **合并门**：`[合并回 main]` 被「所有文件已看过」勾选 + rebase✓ + 测试○ + 评论已解决○ 门控。冲突时自动派合并解决成员，活动流实时显示。

## 6. Composer 输入区

底部全宽、多行（~5 行封顶、自增长），但增强为「驾驶舱指挥台」。沿用现状大部分能力，补上 HITL 控制。

```
┌──────────────────────────────────────────────────────────────┐
│ [📎 login.png ×]                            ← 附件缩略图(可删) │
│ @给谁说… (默认队长)                                            │
│                                                               │
│ ───────────────────────────────────────────────────────────  │
│ [@成员] [/会话动作]   opus·high▾  ✨优化      [打断▾] ↵发送   │
└──────────────────────────────────────────────────────────────┘
   ⏎ 发送 · ⇧⏎ 换行 (桌面)  |  移动端: ⏎ 换行, 显式发送键
```

- **路由 @成员**：默认发队长；`@测试` inline 路由到具体成员（沿用现状 autocomplete，按 role/id 前缀匹配）。**修现状 bug**：草稿在切换到「无队长方向」时，composer 不假装可发——若收件人不可达，禁用并提示「此会话队长已退出，[重启会话]」（治「stale draft 静默 bootstrap」P2）。
- **打断 vs 排队（rCoding 头号坑，必须区分）**：发送键旁 `[打断▾]` 下拉两态——
  - **排队下一轮**（默认 ⏎）：消息作 pending chip 挂在 composer 上方，队长本轮结束后处理。
  - **打断现在**（⌘⏎）：路由到队长，可中止成员、重规划。**绝不静默把纠偏排队**——这是 rCoding 明确点名的最大投诉。
  每个正在跑的成员，在「现在」区有独立 `■停` 微钮（per-member Stop，rPrinciples HITL）。
- **模型选择**：`opus·high▾` pill（tier + reasoning effort，沿用 ModelPicker）。**修现状 P2**：切换前弹确认句「切换会重启队长、打断在途答复 [取消][确认]」，执行后落系统卡，**不再静默 kill-restart**。
- **优化**：`✨优化` 调 `/api/optimizePrompt` 重写草稿，inline 提案 + undo（沿用现状）。**修现状 P2**：图片上传失败时自动从草稿移除失败路径 + 提示，不让人误以为带了附件。
- **附件**：粘贴/拖拽/路径，缩略图可删（沿用现状）。

## 7. 空状态 / 启动中 / 失败状态（诚实，禁假绿点）

贯彻**事实律**：任何状态只描述服务端证实的事实；首个真实输出前绝不出现绿色/「工作中」。

### 启动中（启动清单原地翻转 + 90s 看门狗）
**不假绿点**。会话创建后，对话区落一张**启动清单卡**，逐步消费真实 bootstrap 事件：

乐队栏对应显 `◐启动中`（沿用五态徽章 `◐启动中`，非绿）。任一步失败或 **90s 看门狗**触发（队长无豁免）→ 清单卡**原地翻转**为失败卡，绝不换视图、绝不静默。

### 失败状态（失败卡永不消失）
```
┌ ⚠ AI 还不能开始工作 ────────────────┐
│ Claude Code 未登录                  │
│ 在终端运行: claude /login  [复制]   │
│ [打开终端登录] [换 Codex 重试] [重试]│
│                          [查看日志] │
└─────────────────────────────────────┘
```
- **单一真相源**：修现状「失败卡 + 成员栏双重报错、措辞不一致」P2——失败只在对话区失败卡呈现一次，乐队栏只显 `✕卡住` 状态点 + tooltip，**不再截断报错**（修现状「请在终… 0:00」P1 截断）。
- **永不消失纪律**：失败会话侧栏保持红 `✕`、收件箱保持钉住、卡片可操作直到处理。
- **自动重试封顶**（rAutonomous 防 silent fix-loop）：引擎错误自动重试封顶 2-3 次，然后翻为失败卡，**每次重试都入活动流**，绝不静默循环。`换引擎` 是有界恢复动作。
- **接管/交还**：失败卡可 `[看终端]` → 右面板终端 `[接管]`（暂停成员 + 给键盘，人去修登录）→ `[交还]`（弹「告诉它你改了什么」→ 注入为消息）。这是浏览器侧自助恢复路径。

## 8. 取舍与代价

**牺牲了什么**
- **窄了的对话区**：双区分离让对话不再独占全屏。对**单成员简单任务**（一个 TODO），右面板大半时间稀疏——为并行付的「常驻税」。缓解：单成员时面板可自动折成 64px 脉搏条，等有第二个成员才展开。
- **一次扫视的信息分裂**：「队长说了什么」在左、「成员在做什么」在右，新用户需建立「左读结论、右看过程」的心智。缓解：空态右面板自我说明 + 派工卡 `[查看变更]` 等跨区跳转把两边缝起来。
- **失去 worker 原始流的默认可见性**：默认折叠 worker 细节是 rMultiAgent 共识，但好奇用户多一次点击成本。缓解：钻取一等公民，点成员即进焦点模式。

**对谁不友好**
- **终端原教旨用户**：习惯盯 PTY 看一切的人，会觉得「现在」区的动词翻译是隔了一层。给他们 `⌃▣` 一键直达原始终端兜底。
- **窄屏 (≤1024px) 用户**：双区天然挤；降级为脉搏条 + 浮层，体验弱于宽屏。这是为宽屏驾驶舱体验做的明确取舍。

**最大风险**
1. **结构化事件覆盖率**（现状 P2）：派工卡/交付卡/失败卡这套语法依赖 AgentActivity tailer 粒度。若 CLI 不吐足够结构（如「崩溃 exit code」vs「挂死」难分），部分叙事降级为纯文本系统消息——必须有这个 fallback，否则面板会有「叙事空洞」。
2. **AgentActivity 重连丢失**（后端 P1）：活动流是 WS broadcast-only，重连后 running→ok 转换会丢，且 in-memory ring 重启即没。要兑现「可回放时间线」承诺，**这是必须先补的后端持久化**（见 §9），否则冷加载后面板空白，反而比现状更显眼地撒谎。
3. **首卡 1-2s 延迟兑现不了**：若 tailer 700ms 轮询 + spawn 启动慢导致首卡迟于 3s，「现在」区会有空窗，用户又回到盯绿点。需要 spawn 即同步落一张 `◐启动中` 占位（不等 tailer）。

### 必须新建的后端支撑（现状 P1，不补则面板撒谎）

1. **AgentActivity 持久化 + 冷加载回灌**（后端 P1）：现 ring 是 in-memory、重启即失。新增 `agent_activity` 表持久化 + `GET /api/agent/:id/activity` 走 DB，**重连/冷加载先 REST snapshot 再接 WS**。否则「可回放时间线」承诺落空。
2. **agent 状态转换审计**（后端 P1）：现只存最新 state，无转换日志。面板「现在」区要展示 thinking→idle→error 历史需 `agent_state_events` 持久化。短期可只保 last-N，但冷加载需要。
3. **系统事件持久化 + 入流**（uxDocs P1）：隔离/登录失败/引擎回退/模型切换等必须双写（messages kind=system 或 swarm_events 表）并入流。这是入流律的硬前提——选 messages 表 vs swarm_events 表需半天 spike 定夺（uxDocs 已标）。
4. **派工/交付结构化事件**（现状 P2 覆盖率风险）：BootstrapStage / 派工 / 交付需从「日志」升格为「事件」。be-model 估「80% 是日志升事件」——需 day-1 spike 验证粒度，不足则补新事件类型。
5. **首卡占位即时落地**：spawn 时不等 tailer，直接同步落 `◐启动中` 活动事件，保 1-2s 首卡。

### 落地路径建议
- **P0（接线为主）**：左右双区骨架 + 「现在」区 + 活动流（喂现有 AgentActivity WS）+ 启动清单翻转 + 失败卡永不消失。不依赖新表，先把「不撒谎 + 不 firehose」立住。
- **P1（补持久化）**：AgentActivity / 状态转换 / 系统事件三处持久化，兑现冷加载与回放，把面板从「live-only」升到「重连不空」。
- **P2（深度面板）**：变更 tab 行级评论 + 合并门 + 钻取焦点模式完整化。
